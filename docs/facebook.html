XQuery Facebook Modules Function Reference

Module: Facebook

Summary and Notes

To use the facebook.xqy module in your own XQuery modules, include the following line in your XQuery prolog:

import module "http://marklogic.com/xdmp/facebook" at "/facebook/facebook.xqy";

The library uses the mlfb: namespace, which is not predefined in the server.

The Facebook Platform API uses a REST-like interface. This means that the Facebook method calls are made over the internet by sending HTTP GET or POST requests to the Facebook API REST server.


Setup Instructions

Copy the files from the src folder into some directory on your file system, let's call it /myprojectpath/.

Use the admin UI to point an HTTP server at the /myprojectpath/ directory.

Make sure that your HTTP server is accessible to the outide world (outside of your firewall). Facebook's servers will make HTTP calls to your server. When registering your application, you will provide Facebook with a callback URL for your application.

Within the settings for the application, be sure to set the following fields (Facebook's websites describes the same process in steps #1 through #8 a the following URL http://developers.facebook.com/step_by_step.php#hello ):

  • Before using the API, the developer must create a new application using the Facebook Developer application found here: http://www.facebook.com/developers/editapp.php?new.
  • The Callback URL should be set to the URL of the directory containing index.xqy on the HTTP server containing the application. Be sure to include the trailing slash ("/") at the end of the URL. Example: http://facebook.demo.marklogic.com:8099/sample/
  • Be sure to set the Canvas Page URL and the Developer Contact E-mail.
  • The check box for "Can your application be added on Facebook?" should be checked.
  • The Post Add URL and Side Nav URL should be set to the Canvas Page URL (ex: http://apps.facebook.com/yourappnamehere/ - where "yourappnamehere" is the name specified for the Canvas Page URL field within the same page). For this URL, not specifying a file name at the end of the URL implies the usage of index.xqy. When we visit http://apps.facebook.com/yourappnamehere/, we are actually visiting http://apps.facebook.com/yourappnamehere/index.xqy.
  • For the field "Who can add your application to their Facebook account?", check the checkbox for "Users" and select "All Pages" for the radio selection.
  • If you desire to keep the application private through development, be sure to check the box for "Developer Mode" so that only developers for the application can view the application.

Edit the API Key and Secret Key as the variables $app-api-key and $app-secret (at the top of all files in the /myprojectpath/sample/ directory) to match the API Key and Secret Key provided for your application through the Developer application.

And you're done! Visit http://apps.facebook.com/yourappnamehere/ to install the sample application and use it.


What's Included

There are two folders found in the src folder: facebook and sample.

The facebook folder contains only one file (facebook.xqy) which is the only module necessary to import in order to use the Facebook platform.

The facebook folder contains three files (index.xqy, sample-basic.xqy, and sample-extended.xqy). Each of these files are examples of sample canvas pages for your application and can be accessed by visiting the URLs http://apps.facebook.com/yourappnamehere/index.xqy (which is equivalent to visiting http://apps.facebook.com/yourappnamehere/), http://apps.facebook.com/yourappnamehere/sample-basic.xqy, and http://apps.facebook.com/yourappnamehere/sample-extended.xqy respectively.

The file index.xqy is an extremely simple Hello World application canvas page. The code in this file creates an application configuration, redirects the user to login if necessary, displays the user's name with FBML, and displays up to 25 of the user's friends. Equivalent PHP Hello World code (provided by Facebook whenever you register an application) can be found commented out at the end of the file.

The code in the file sample-basic.xqy does everything that the code in index.xqy does except that it also has a few best practices. The code redirects the user if they have not added the application (adding an application is a separate process from logging into an application - Facebook claims that it will be merging these two processes into one some time in the middle of 2008). The code also uses a try-catch statement to catch exceptions and sets the user's profile box.

The code in the file sample-extended.xqy does everything that the code in sample-basic.xqy does except that it also calls a bunch of additional functions from the library, such as displaying the user's events and creating a feed item.


XQuery Facebook Platform Functions

All functions that map to functions specified in the Facebook API shall make up to five attempts to retry the method if the connection the Facebook server is dropped.

Some methods available through the Facebook Platform API do not have an equivalent XQuery Facebook Library function. These methods are either deprecated, in Beta, newly released, or not typically used (such as the Market-place methods). Any function correlating to a platform API method that is not available through the library can be written by using the function mlfb:post-method. Please view the code for any of the library functions that utilize a Facebook Platform API method for assistance in writing XQuery functions for any method without a corresponding function. We encourage you to add new functions to the project for this library.

Support for iframes and for cookies is not currently built into the library.


Naming Convention

The names for all library functions that mirror functions from the Facebook Platform API have been converted from camel-style to words separated by dashes. Other than the style, the words making up the functions are taken from the Facebook Platform API found at the Facebook API reference page.

Any function arguments marked as "optional" must be set to the variable $mlfb:null if the user does not want to use them. Every Facebook API function designated to return a type node() will return an XML node in the formats specified by the Facebook Platform API found here: the Facebook API reference page.
